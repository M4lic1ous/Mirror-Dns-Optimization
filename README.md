# Mirror-Dns-Optimization
# 🚀 بهینه‌ساز Mirror و DNS برای Kali Linux

این پروژه یک اسکریپت Bash است که به‌صورت خودکار بهترین Mirror و سریع‌ترین DNS را شناسایی و روی سیستم اعمال می‌کند تا سرعت به‌روزرسانی بسته‌ها و کیفیت اتصال شبکه بهبود پیدا کند.

## ✨ قابلیت‌ها

* ✅ تشخیص خودکار نسخه و نام (Codename) سیستم‌عامل.
* ✅ بررسی و انتخاب سریع‌ترین Mirror از بین آدرس‌های موجود در `mirror.txt`.
* ✅ اعتبارسنجی Mirrorها قبل از استفاده برای جلوگیری از اعمال مخازن نامعتبر.
* ✅ اندازه‌گیری عملکرد Mirrorها بر اساس زمان پاسخ و سرعت دانلود فایل‌های واقعی مخزن.
* ✅ بررسی و انتخاب بهترین DNS از فایل `dns.txt` بر اساس کمترین تأخیر (Latency).
* ✅ پشتیبانی از روش‌های مختلف تنظیم DNS مانند:

  * `resolvectl`
  * `systemd-resolve`
  * `NetworkManager`
  * `resolvconf`
  * ویرایش مستقیم `resolv.conf`
* ✅ تهیه نسخه پشتیبان از فایل‌های مهم سیستم قبل از هرگونه تغییر.
* ✅ بررسی و نصب خودکار وابستگی‌های موردنیاز در صورت امکان.
* ✅ نمایش رابط کاربری رنگی، نوار پیشرفت و اطلاعات مرحله‌به‌مرحله در ترمینال.
* ✅ ثبت گزارش کامل عملیات در فایل لاگ.
* ✅ امکان بازگردانی تنظیمات قبلی از طریق فایل‌های پشتیبان.

## 📂 فایل‌های موردنیاز

### `mirror.txt`

لیستی از Mirrorهای کالی لینوکس، هر آدرس در یک خط.

نمونه:

```text
https://http.kali.org/kali/
https://mirror.example.com/kali/
```

### `dns.txt`

لیستی از DNS Serverها، هر IP در یک خط.

نمونه:

```text
1.1.1.1
8.8.8.8
9.9.9.9
```

## ▶️ نحوه اجرا

ابتدا فایل را قابل اجرا کنید:

```bash
chmod +x bash.sh
```

سپس با دسترسی روت اجرا کنید:

```bash
sudo ./bash.sh
```

## ⚙️ روند عملکرد اسکریپت

1. شناسایی اطلاعات سیستم‌عامل.
2. تنظیم اولیه مخزن برای نصب وابستگی‌های موردنیاز.
3. بررسی ابزارهای لازم و نصب آن‌ها در صورت نیاز.
4. خواندن Mirrorها از `mirror.txt`.
5. اعتبارسنجی و تست عملکرد هر Mirror.
6. انتخاب بهترین Mirror و اعمال آن روی `sources.list`.
7. خواندن DNSها از `dns.txt`.
8. اندازه‌گیری زمان پاسخ هر DNS.
9. انتخاب سریع‌ترین DNS و اعمال آن روی سیستم.
10. ایجاد نسخه پشتیبان و ثبت گزارش عملیات.

## 💾 فایل‌های پشتیبان

قبل از هر تغییر، فایل‌های زیر با تاریخ و زمان ذخیره می‌شوند:

```text
/etc/apt/sources.list.backup.YYYYMMDD_HHMMSS
/etc/resolv.conf.backup.YYYYMMDD_HHMMSS
```

برای بازگردانی تنظیمات قبلی می‌توانید از دستورات زیر استفاده کنید:

```bash
cp /etc/apt/sources.list.backup.TIMESTAMP /etc/apt/sources.list
cp /etc/resolv.conf.backup.TIMESTAMP /etc/resolv.conf
```

## 📝 فایل لاگ

تمام رویدادها و نتایج اجرا در مسیر زیر ذخیره می‌شوند:

```text
/var/log/mirror_optimizer.log
```

## ⚠️ نکات مهم

* اجرای اسکریپت نیازمند دسترسی **Root** یا `sudo` است.
* این پروژه عمدتاً برای **Kali Linux** طراحی شده است.
* کیفیت انتخاب Mirror و DNS به محتوای فایل‌های `mirror.txt` و `dns.txt` بستگی دارد.
* شرایط شبکه و موقعیت جغرافیایی می‌توانند روی نتایج تست‌ها تأثیر بگذارند.
* در برخی سیستم‌ها، سرویس‌های مدیریت شبکه ممکن است تنظیمات DNS را مجدداً بازنویسی کنند.
